# 论利用快速傅里叶变换（FFT）求解 LeetCode 396 题

本文旨在论述如何通过代数与信号处理的视角，将 LeetCode 396 题（旋转函数）转化为循环卷积（Circular Convolution）问题，并利用快速傅里叶变换（FFT）进行求解。同时，将对该算法的时间复杂度进行理论推导。

---

## 1. 问题的数学建模

给定长度为 $n$ 的数组 $A = \text{nums}$。旋转函数 $F(k)$ 定义为数组顺时针旋转 $k$ 次后，元素与其对应索引乘积之和。

数组顺时针旋转 $k$ 位后，当前处于第 $i$ 个位置的元素，其在原数组 $A$ 中的索引为 $(i - k) \bmod n$。因此，$F(k)$ 可以表示为：
$$F(k) = \sum_{i=0}^{n-1} i \cdot A_{(i - k) \bmod n}$$

为构造标准卷积形式，引入权重序列 $W = [0, 1, 2, \dots, n-1]$，即 $W_i = i$。同时定义序列 $B$，使得 $B_i = A_{-i \bmod n}$。该操作等价于保留 $A_0$ 位置不变，将其余元素反转。代入原式可得：
$$F(k) = \sum_{i=0}^{n-1} W_i \cdot B_{(k - i) \bmod n}$$

上述公式符合循环卷积的数学定义：$F = W \circledast B$。因此，求解 $F(k)$ 的过程可转化为计算序列 $W$ 和序列 $B$ 循环卷积后的第 $k$ 个元素。

---

## 2. 傅里叶变换与卷积定理的应用

若在时域内直接计算循环卷积 $F = W \circledast B$，需要进行 $n$ 次滑动计算，每次包含 $n$ 个元素的乘加运算，其时间复杂度为 $O(n^2)$。在 $n = 10^5$ 的数据规模下会导致计算超时。

在此引入卷积定理（Convolution Theorem）。该定理指出，时域中的卷积等价于频域中的逐元素相乘：
$$\text{FFT}(F) = \text{FFT}(W) \times \text{FFT}(B)$$

基于此定理，计算流程可优化为以下步骤：
1. 通过 FFT 将序列 $W$ 和 $B$ 转换至频域。
2. 在频域中执行逐元素相乘运算，此时乘法操作的时间复杂度降至 $O(n)$。
3. 通过逆快速傅里叶变换（IFFT）将结果转换回时域。

通过这一数学转换，算法规避了时域内 $O(n^2)$ 的运算过程。

---

## 3. 时间复杂度理论推导

标准离散傅里叶变换（DFT）的公式为：
$$X[k] = \sum_{n=0}^{N-1} x[n] \cdot W_N^{kn}$$
直接计算 DFT 的复杂度同样为 $O(n^2)$。NumPy 底层采用的 Cooley-Tukey 算法通过分治策略优化了该过程。

假设数组长度为 $N$，按索引的奇偶性将原序列分为两组。设 $E[k]$ 为偶数项的 DFT，$O[k]$ 为奇数项的 DFT。利用旋转因子 $W_N$ 的周期性与对称性，可将原式重组为：
$$X[k] = E[k] + W_N^k \cdot O[k]$$
$$X\left[k + \frac{N}{2}\right] = E[k] - W_N^k \cdot O[k]$$

该过程称为蝶形运算。计算前半部分后，仅需一次加减法即可得出后半部分的结果。因此，长度为 $N$ 的 DFT 被分解为两个长度为 $N/2$ 的 DFT，以及一次时间复杂度为 $O(N)$ 的线性合并操作。

其时间复杂度的递归关系式为：
$$T(N) = 2T\left(\frac{N}{2}\right) + O(N)$$
根据主定理展开，递归深度为 $\log_2 N$，单层合并工作量为 $O(N)$，因此单次 FFT 的时间复杂度为 $O(N \log N)$。

---

## 4. 算法各阶段复杂度总结

将整体求解过程分解，各阶段的复杂度如下：

1. **序列构造（反转数组）：** 构造序列 $W$ 和 $B$，耗时 $O(n)$。
2. **频域转换：** 执行两次 FFT 操作，耗时 $2 \times O(n \log n)$。
3. **频域相乘：** 两个频域数组逐元素相乘，耗时 $O(n)$。
4. **时域恢复：** 执行一次 IFFT 操作，耗时 $O(n \log n)$。
5. **数据后处理：** 取实部、消除浮点误差并求最大值，耗时 $O(n)$。

综上所述，基于循环卷积与 FFT 的解法整体性能如下：
* **时间复杂度：** $O(n \log n)$
* **空间复杂度：** $O(n)$ （主要源于频域复数数组的内存开销）

## 5. Python 代码实现

```python
import numpy as np

class Solution:
    def maxRotateFunction(self, nums: list[int]) -> int:
        n = len(nums)
        if n == 0:
            return 0
        
        # 1. 构造序列 W: [0, 1, 2, ..., n-1]
        W = np.arange(n)
        
        # 2. 构造序列 B: B[i] = nums[-i mod n]
        B = np.zeros(n)
        B[0] = nums[0]
        if n > 1:
            B[1:] = nums[:0:-1] 
            
        # 3. 计算 W 和 B 的快速傅里叶变换
        fft_W = np.fft.fft(W)
        fft_B = np.fft.fft(B)
        
        # 4. 频域下逐元素相乘
        fft_F = fft_W * fft_B
        
        # 5. 逆傅里叶变换
        F = np.fft.ifft(fft_F).real
        
        # 消除浮点数精度误差并转换为整数
        F = np.round(F).astype(int)
        
        # 6. 返回结果最大值
        return int(np.max(F))
