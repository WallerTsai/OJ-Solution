这道题如果直接使用暴力双层循环枚举所有的 $i$ 和 $j$，时间复杂度将是 $O(n^2)$。对于 $n = 10^5$ 的数据规模，计算量达到 $10^{10}$ 次，绝对会超时（TLE）。

处理这种包含“位运算（异或、与、或）”以及“求和”的题目，有一个非常经典且通用的降维打击技巧：**拆位法（按位计算）** 结合 **前缀和思想**。

下面我们一步步来拆解这道题的满分逻辑。

### 1. 数学推导与拆位思想

异或运算 $\oplus$ 是按位独立的。也就是说，数字 $A$ 和 $B$ 的第 $k$ 位异或结果，绝对不会影响第 $k+1$ 位。因此，我们可以把题目中的公式拆成对每一位单独求和：

$$\sum_{i=1}^{n} \sum_{j=i+1}^{n} (a_i \oplus a_j) \times (j - i)$$

设 $a_i$ 的第 $k$ 个二进制位为 $a_i^{(k)}$（取值为 0 或 1），那么 $a_i \oplus a_j = \sum_{k=0}^{19} (a_i^{(k)} \oplus a_j^{(k)}) \times 2^k$。
（因为 $a_i \le 2^{20}$，所以二进制最多只有 20 位，即 $k \in [0, 19]$）。

我们将求和顺序交换，先遍历每一位 $k$：
$$\text{Ans} = \sum_{k=0}^{19} 2^k \left( \sum_{i=1}^{n} \sum_{j=i+1}^{n} (a_i^{(k)} \oplus a_j^{(k)}) \times (j - i) \right)$$

### 2. $O(n)$ 求解单位的贡献

现在，我们单独看某一个特定的第 $k$ 位。在这个维度下，所有的 $a_i$ 都变成了 $0$ 或 $1$。
什么样的数对 $(a_i^{(k)}, a_j^{(k)})$ 异或之后才会产生贡献呢？
**只有当一个是 0，另一个是 1 时**，异或结果才为 $1$，才会把距离 $(j - i)$ 加到总和里。

所以，我们的目标变成了：对于每一位，求出所有**值不相等**的下标对 $(i, j)$ 的距离和 $(j - i)$。

如果你在从左到右遍历数组到第 $j$ 个元素（假设它的第 $k$ 位是 $1$），那么它和前面所有第 $k$ 位是 $0$ 的元素（假设下标为 $i_1, i_2 \dots$）都能组成合法对。它对答案的贡献是：
$$(j - i_1) + (j - i_2) + \dots + (j - i_m)$$
提取公因式 $j$，等价于：
$$j \times m - (i_1 + i_2 + \dots + i_m)$$

其中：
* $m$ 是当前遍历到位置 $j$ 时，前面出现过的 $0$ 的**个数**。
* $(i_1 + i_2 + \dots + i_m)$ 是前面出现过的 $0$ 的**下标之和**。

同理，如果当前第 $j$ 位的元素是 $0$，它就能和前面出现过的 $1$ 组成合法对，计算公式一样，只需把对 $0$ 的统计换成对 $1$ 的统计。

### 3. Python 满分代码实现

根据上面的推导，我们只需遍历 $20$ 个位，每次对数组进行一次 $O(n)$ 的线性扫描。整体复杂度为 $O(20 \times n)$，完美通过 $10^5$ 的数据。

```python
import sys

# 优化读入速度
input = sys.stdin.readline

def main():
    n = int(input())
    if n == 0:
        return
    a = list(map(int, input().split()))
    
    total_ans = 0
    
    # 题目约定 ai <= 2^20，所以我们遍历 0 到 19 位
    for k in range(20):
        # 分别记录目前遇到的 0 和 1 的个数
        count_0 = 0
        count_1 = 0
        # 分别记录目前遇到的 0 和 1 的下标之和
        sum_idx_0 = 0
        sum_idx_1 = 0
        
        # 记录当前这一位 (第 k 位) 对结果的贡献总和
        current_bit_sum = 0
        
        # 遍历所有数字，注意题目里的下标 i 和 j 都是从 1 开始计数的
        for j in range(1, n + 1):
            val = a[j - 1]
            
            # 判断当前数字 val 的第 k 位是 0 还是 1
            if (val >> k) & 1:
                # 如果当前位是 1，它与前面所有的 0 产生贡献
                current_bit_sum += j * count_0 - sum_idx_0
                # 更新 1 的统计信息
                count_1 += 1
                sum_idx_1 += j
            else:
                # 如果当前位是 0，它与前面所有的 1 产生贡献
                current_bit_sum += j * count_1 - sum_idx_1
                # 更新 0 的统计信息
                count_0 += 1
                sum_idx_0 += j
                
        # 将当前这一位的总贡献乘上它的位权 (2^k)，累加到最终答案里
        total_ans += current_bit_sum << k

    print(total_ans)

if __name__ == '__main__':
    main()
```

### 复杂度分析
* **时间复杂度**：外层循环 20 次，内层循环 $n$ 次。总执行次数约为 $2 \times 10^6$ 次运算。在 Python 中通常只需要 $0.1 \sim 0.3$ 秒，完全在 $1.0$ 秒的时间限制内。
* **空间复杂度**：$O(n)$，只需要存储原数组 `a`，占用极少内存。

这是一个非常优美且经典的 O(N) 算法，以后遇到形如 $\sum (A_i \oplus A_j)$ 或者按位求距离的题目，直接无脑上**拆位统计法**即可。
