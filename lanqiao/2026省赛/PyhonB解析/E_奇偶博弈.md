## 蓝桥杯《奇偶博弈》深度解析

本题是一个典型的**公平组合游戏 (Impartial Game)**。解决这类问题的万能钥匙是 **Sprague-Grundy (SG) 定理**。

### 一、 核心工具：MEX 运算与 SG 值

在博弈论中，我们通过以下两个概念将复杂的规则抽象化：

#### 1. MEX 运算 (Minimum Excluded value)
**定义**：找到一个非负整数集合中，**最小的、未出现过**的非负整数。
* $MEX(\{1, 2, 3\}) = 0$
* $MEX(\{0, 1, 3\}) = 2$
* **博弈意义**：它代表了一个状态所能到达的所有“出路”中，缺掉的那个最小分数。



#### 2. SG 值 (Sprague-Grundy Value)
**定义**：$SG(x) = MEX(\{ SG(y) \mid x \to y \})$，其中 $y$ 是 $x$ 的后继状态。
* **$SG(x) = 0$**：**必败态**。你接手这个状态时，无论怎么走，对方都能赢。
* **$SG(x) > 0$**：**必胜态**。你一定能一步把它变成 $SG=0$ 的状态留给对手。

---

### 二、 针对本题的 SG 值推导

题目给出了两种操作规则，我们分别计算其 SG 值：

#### 1. 偶数状态 $x$ 的 SG 值
偶数可以变成 $\{0, 2, 4, \dots, x-2\}$ 中的任意偶数。
* $SG(0) = 0$ （终点，无法操作）
* $SG(2) = MEX(\{SG(0)\}) = MEX(\{0\}) = 1$
* $SG(4) = MEX(\{SG(0), SG(2)\}) = MEX(\{0, 1\}) = 2$
* **结论**：对于偶数 $x$，$SG(x) = x / 2$。



#### 2. 奇数状态 $x$ 的 SG 值（解题关键）
奇数 $x$ **只能**变成 $x-1$（唯一的后继状态）。
* **当 $x = 1$ 时**：$SG(1) = MEX(\{SG(0)\}) = MEX(\{0\}) = 1$。
* **当 $x \ge 3$ 时**：$SG(x) = MEX(\{SG(x-1)\})$。
    根据上面的推导，因为 $x-1$ 是 $\ge 2$ 的偶数，所以 $SG(x-1) \ge 1$。
    由于集合 $\{SG(x-1)\}$ 里面**没有 0**，所以 $MEX$ 运算的结果一定是 **0**。
* **结论**：
    * 若 $W_i = 1$，$SG(W_i) = 1$。
    * 若 $W_i \ge 3$，$SG(W_i) = 0$。

---

### 三、 胜负判定逻辑：异或和与简化

当游戏由 $N$ 个独立的数字组成时，总局面的 SG 值为：
$$Total\_SG = SG(W_1) \oplus SG(W_2) \oplus \dots \oplus SG(W_N)$$

**根据题目限定：初始所有 $W_i$ 均为正奇数。**
这意味着：
1.  所有大于 1 的奇数（3, 5, 7...），它们的 SG 值都是 **0**。在异或运算中，0 不会改变任何结果。
2.  只有数字 **1**，它的 SG 值是 **1**。



**最终简化策略：**
整个游戏的胜负，仅仅取决于数组中数字 **1** 出现的次数：
* 如果 `1` 的个数是**奇数**：$Total\_SG = 1$（必胜态），输出 **L** (小蓝胜)。
* 如果 `1` 的个数是**偶数**：$Total\_SG = 0$（必败态），输出 **Q** (小桥胜)。

---

### 四、 Python 过题代码实现

由于逻辑已经简化为“数 1 的个数”，代码极其高效。

```python
import sys

def solve():
    # 采用标准输入一次性读取，应对 2*10^5 级别的数据量
    input_data = sys.stdin.read().split()
    if not input_data:
        return
    
    T = int(input_data[0])
    idx = 1
    results = []
    
    for _ in range(T):
        N = int(input_data[idx])
        idx += 1
        # 获取当前组的 N 个数字（以字符串形式）
        arr = input_data[idx : idx + N]
        idx += N
        
        # 核心：统计字符串 '1' 出现的次数
        # 即使数字很大（如 10^9），我们也只关心它是不是 '1'
        count_ones = arr.count('1')
        
        if count_ones % 2 == 1:
            results.append("L")
        else:
            results.append("Q")
            
    # 批量输出，提高 I/O 效率
    sys.stdout.write('\n'.join(results) + '\n')

if __name__ == '__main__':
    solve()
```

### 总结
这道题的精妙之处在于：它给出了复杂的偶数操作规则作为“烟雾弹”，但通过**初始状态全是奇数**这一限制，使得所有大于 1 的奇数在博弈的第一步就跳过了 $SG=0$（必败态）的出路，从而使它们自己沦为 $SG=0$ 的状态。解题者只要看穿这一点，就能将难题瞬间化简。
